# 多图输入节点使用说明

## 节点概述

**Multi-Image Input (Story Creation)** 节点支持两种工作模式：

1. **图像模式**：分析多张图片并创作故事内容，支持多种视频生成模型（WAN2.2、LTX2等）
2. **文本模式**：通过选项设置生成提示词，无需图片输入

## 节点功能

### 主要功能
1. **双模式支持**：图片模式和文本模式灵活切换
2. **多图输入**：支持输入多张图片（图片模式）
3. **自动预处理**：自动缩放和编码图片
4. **故事创作**：生成适合视频生成的故事内容
5. **灵活配置**：支持多种故事类型、长度和风格
6. **多种应用**：支持故事创作、剧本创作、广告文案等多种内容类型
7. **图片输出**：将图片数据传递给推理节点

### 输入参数

#### 工作模式
- **mode** (下拉选择): 工作模式
  - 图像模式：分析图片内容进行故事创作
  - 文本模式：通过选项设置生成提示词

#### 图片输入（仅图像模式）
- **image1** (IMAGE): 第一张输入图片
- **image2** (IMAGE): 第二张输入图片
- **image3** (IMAGE): 第三张输入图片
  - 可以只连接一张或两张图片
  - 至少需要提供一张图片（图片模式）
  - 自动检测图片数量
  - 自动预处理和编码

#### 配置参数
- **story_type** (下拉选择): 内容创作类型
  - 连贯故事：创作一个连贯完整的故事
  - 分镜描述：按照时间顺序描述每个场景
  - 场景分析：深入分析每个场景的细节
  - 角色发展：描述角色的成长和变化
  - 情感递进：展现情感的变化和发展
  - 创意写作：进行创意写作，展现独特的想法和创意
  - 剧本创作：创作一个完整的剧本，包含对话和场景描述
  - 广告文案：创作吸引人的广告文案
  - 产品介绍：撰写详细的产品介绍
  - 教育内容：创作教育性内容，易于理解和学习

- **story_length** (下拉选择): 内容长度
  - 简短（200字以内）：快速、简洁的内容
  - 中等（400字以内）：平衡的内容长度
  - 详细（600字以内）：丰富的内容
  - 完整（1000字以内）：完整的内容叙述

- **language** (下拉选择): 输出语言
  - 中文：生成中文内容
  - English：生成英文内容

- **max_size** (整数): 图片最大尺寸（像素）
  - 范围：128-512
  - 默认：256
  - 用于控制图片缩放（仅图像模式）

- **custom_prompt** (文本): 自定义提示词
  - 用于指导内容创作
  - 可以添加特定的要求或风格
  - 多行文本输入

- **include_image_descriptions** (布尔值): 是否包含图片描述
  - True：在故事前包含每张图片的描述
  - False：直接创作故事，不包含图片描述
  - 仅图像模式有效

- **story_theme** (下拉选择): 内容主题
  - 无特定主题：不限制主题
  - 冒险故事：冒险主题，充满挑战和探索
  - 浪漫故事：浪漫主题，展现情感和关系
  - 悬疑故事：悬疑主题，设置悬念和谜题
  - 科幻故事：科幻主题，展现未来科技
  - 奇幻故事：奇幻主题，包含魔法和超自然元素
  - 日常生活：日常生活主题，展现平凡中的美好
  - 历史故事：历史主题，展现过去的文化和时代
  - 未来科技：未来科技主题，展现先进科技和人类发展
  - 商业营销：商业营销主题，突出产品或服务优势
  - 教育科普：教育科普主题，传播知识和信息
  - 娱乐搞笑：娱乐搞笑主题，带来欢乐和轻松

- **narrative_style** (下拉选择): 叙事风格
  - 第一人称：使用第一人称视角（我）
  - 第三人称：使用第三人称视角（他/她）
  - 全知视角：使用全知视角，了解所有角色的想法
  - 多视角切换：在不同角色之间切换视角

- **content_focus** (下拉选择): 内容重点
  - 平衡发展：平衡发展情节、角色、情感等各个方面
  - 强调情节：重点突出情节的发展和转折
  - 强调角色：重点刻画角色的性格和成长
  - 强调情感：重点展现情感的变化和表达
  - 强调视觉：重点描述视觉元素和场景细节
  - 强调对话：重点通过对话推动故事发展

- **target_audience** (下拉选择): 目标受众
  - 普通大众：适合普通大众阅读和理解
  - 青少年：适合青少年阅读，内容积极向上
  - 儿童：适合儿童阅读，语言简单易懂
  - 专业人士：适合专业人士阅读，内容专业深入
  - 特定群体：适合特定群体阅读，针对性强

- **video_model** (下拉选择): 视频生成模型类型
  - WAN2.2：WAN2.2视频生成模型，强调场景描述和视觉元素，使用简洁有力的语言
  - LTX2：LTX2视频生成模型，注重细节描述和情感表达，突出关键场景转换
  - General Video：通用视频生成模型，平衡场景描述和叙事流畅性
  - Custom：自定义视频生成模型，根据具体模型要求调整

### 输出参数

- **prompt** (STRING): 生成的内容创作提示词
  - 图片模式：包含图片分析要求
  - 文本模式：包含详细的创作指导
  - 适合直接连接到llama_cpp_instruct_adv节点的custom_prompt

- **images** (IMAGE): 图片数据
  - 图片模式：返回预处理后的图片数据
  - 文本模式：返回None
  - 适合直接连接到llama_cpp_instruct_adv节点的images输入

## 使用方法

### 图片模式使用方法

#### 基本工作流

```
1. 准备图片（1-6张）
   ↓
2. Multi-Image Input节点（图片模式）
   ↓
3. Llama-cpp Image Inference节点
   ↓
4. 获取故事内容
   ↓
5. 用于视频生成（WAN2.2、LTX2等）
```

#### 详细步骤

1. **准备图片**：
   - 准备1-6张图片（至少1张）
   - 确保图片内容连贯，有故事性
   - 图片可以是连续的场景或不同场景的片段

2. **配置Multi-Image Input节点**：
   - 选择`mode`为"图像模式"
   - 连接图片输入到`image1`到`image6`端口（至少连接一个）
   - 选择`story_type`（推荐"连贯故事"或"分镜描述"）
   - 选择`story_length`（通常需要400字以内）
   - 选择`language`（中文或英文）
   - 选择`video_model`（WAN2.2/LTX2/General Video/Custom）
   - 调整其他参数（主题、叙事风格等）

3. **连接到Llama-cpp Image Inference节点**：
   - 将`prompt`输出连接到`custom_prompt`输入
   - 将`images`输出连接到`images`输入
   - 在Llama-cpp Image Inference节点中：
     - `inference_mode`选择"images"
     - `preset_prompt`选择"[Creative] Short Story"
     - `output_language`选择与Multi-Image Input相同的语言

4. **获取故事内容**：
   - 运行工作流
   - 等待模型处理
   - 获取生成的故事内容

5. **用于视频生成**：
   - 根据选择的`video_model`类型，将故事内容复制到对应的视频生成模型（WAN2.2、LTX2等）的提示词输入
   - 根据故事内容调整视频参数
   - 生成视频

### 文本模式使用方法



#### 详细步骤

1. **配置Multi-Image Input节点**：
   - 选择`mode`为"文本模式"
   - 不需要连接任何图片
   - 选择`story_type`（根据需求选择）
   - 选择`story_length`（根据内容长度需求）
   - 选择`language`（中文或英文）
   - 配置其他参数（主题、叙事风格、内容重点、目标受众）
   - 可选：添加`custom_prompt`自定义要求

2. **连接到Llama-cpp Image Inference节点**：
   - 将`prompt`输出连接到`custom_prompt`输入
   - 不需要连接`images`输出（文本模式返回None）
   - 在Llama-cpp Image Inference节点中：
     - `inference_mode`选择"text"
     - `preset_prompt`选择"[Creative] Short Story"或其他合适预设
     - `output_language`选择与Multi-Image Input相同的语言

3. **获取创作内容**：
   - 运行工作流
   - 等待模型处理
   - 获取生成的内容

4. **使用生成内容**：
   - 用于视频生成（WAN2.2、LTX2等）
   - 用于其他文本生成应用
   - 用于创意写作或内容创作

## 最佳实践

### 模式选择建议
1. **图像模式**：
   - 有具体的图片素材需要分析
   - 需要根据图片内容创作故事
   - 图片内容连贯，有故事性

2. **文本模式**：
   - 没有图片素材，需要从零创作
   - 需要快速生成内容
   - 需要灵活控制内容类型和风格

### 图片选择建议（仅图像模式）
1. **连贯性**：选择内容连贯的图片，有助于创作更好的故事
2. **数量适中**：建议1-6张图片，根据需要选择合适的数量
3. **质量优先**：使用高质量的图片，有助于模型理解
4. **场景多样**：包含不同的场景和角度，丰富故事内容

### 参数设置建议
1. **工作模式**：
   - 有图片素材：选择图片模式
   - 无图片素材：选择文本模式

2. **内容类型**：
   - WAN2.2视频生成：推荐"连贯故事"或"分镜描述"
   - LTX2视频生成：推荐"角色发展"或"情感递进"
   - 通用视频生成：推荐"连贯故事"或"场景分析"
   - 创意写作：推荐"角色发展"或"情感递进"
   - 商业应用：推荐"广告文案"或"产品介绍"
   - 教育应用：推荐"教育内容"

3. **内容长度**：
   - 短视频（10-30秒）：200字以内
   - 中等视频（30-60秒）：400字以内
   - 长视频（60-120秒）：600字以内
   - 短篇内容：200字以内
   - 中篇内容：400字以内
   - 长篇内容：600字以内

4. **叙事风格**：
   - 第一人称：适合个人化的故事
   - 第三人称：适合客观的描述
   - 全知视角：适合复杂的故事
   - 多视角切换：适合多角色故事

5. **内容重点**：
   - 平衡发展：适合大多数场景
   - 强调情节：适合快节奏内容
   - 强调角色：适合人物驱动的故事
   - 强调情感：适合情感类内容
   - 强调视觉：适合视频生成
   - 强调对话：适合剧本创作

## 常见问题

### Q1：图像模式和文本模式有什么区别？
**A**：
- **图像模式**：需要提供图片，模型会分析图片内容并基于图片创作故事
- **文本模式**：不需要图片，通过选项设置生成提示词，模型根据提示词创作内容
- **选择建议**：有图片素材用图像模式，无图片素材用文本模式

### Q2：为什么模型输出的故事不够连贯？
**A**：可能的原因：
1. 图片内容不够连贯（图像模式）
2. 故事类型选择不当
3. 自定义提示词不够明确
4. 参数设置不合理

**解决方案**：
1. 选择内容更连贯的图片（图像模式）
2. 尝试不同的故事类型
3. 添加更具体的自定义提示词
4. 调整参数设置（内容重点、叙事风格等）

### Q3：如何提高故事质量？
**A**：建议：
1. **图像模式**：
   - 使用高质量的图片
   - 选择合适的故事长度
   - 添加具体的自定义提示词
   - 选择适合的故事主题和叙事风格

2. **文本模式**：
   - 详细设置参数（类型、长度、主题、风格等）
   - 添加具体的自定义提示词
   - 选择适合的内容重点和目标受众
   - 合理配置叙事风格

### Q4：文本模式可以用于哪些场景？
**A**：文本模式适用于：
1. 创意写作：小说、短篇故事、诗歌
2. 剧本创作：电影剧本、广告剧本、短视频脚本
3. 商业文案：广告文案、产品介绍、营销文案
4. 教育内容：教学材料、科普文章、知识讲解
5. 内容生成：社交媒体内容、博客文章、新闻稿

### Q5：视频生成效果不好？
**A**：可能的原因：
1. 故事内容不够具体
2. 故事长度不合适
3. 语言不匹配
4. 内容重点选择不当
5. 视频模型选择不当

**解决方案**：
1. 优化故事内容，使其更具体
2. 调整故事长度以匹配视频长度
3. 确保语言与视频内容一致
4. 选择"强调视觉"的内容重点
5. 使用图像模式，提供视觉参考
6. 根据目标视频生成模型选择合适的`video_model`参数：
   - WAN2.2：强调场景描述和视觉元素，使用简洁有力的语言
   - LTX2：注重细节描述和情感表达，突出关键场景转换
   - General Video：平衡场景描述和叙事流畅性
   - Custom：根据具体模型要求调整

### Q6：如何正确连接图片数据到推理节点？
**A**：
1. **图像模式**：
   - 将`prompt`输出连接到`custom_prompt`输入
   - 将`images`输出连接到`images`输入
   - 确保两个连接都正确建立

2. **文本模式**：
   - 将`prompt`输出连接到`custom_prompt`输入
   - 不需要连接`images`输出（文本模式返回None）
   - 只需要连接prompt输出
